# 外部接入准入规范（工具与开源项目）

目标：提升开发效率，同时控制依赖风险与技术债。

---

## 适用范围

- 外部工具（项目管理、监控、分析、CI/CD）
- GitHub 开源项目（插件、系统模板、工具脚本）
- 第三方资产（美术、音频、特效）

---

## 准入五原则（必须全部满足）

1. **许可证清晰**：允许当前项目使用场景（含商业化可能）。
2. **维护活跃**：近 3-6 个月有有效提交或活跃 issue。
3. **文档可用**：有快速上手、核心 API/配置说明。
4. **替换可行**：若弃用可在 1-2 天内替换或回退。
5. **收益可证**：7 天内能量化效率或质量收益。

---

## 接入流程（固定）

1. 候选登记：记录来源、用途、风险、预估收益。
2. 隔离验证：在独立分支/模块 Demo 先接入。
3. 门禁验证：运行现有验证脚本，确认无主链路破坏。
4. 回退演练：验证“移除后项目仍可运行”。
5. 决策归档：保留或放弃，并写明原因。

---

## 风险分级

- **高风险**：改动主链路核心系统、未知许可证、深度耦合
- **中风险**：影响性能路径、涉及关键数据结构
- **低风险**：纯工具层、可配置开关、可快速回滚

高风险接入必须附带：
- 回退步骤
- 影响面说明
- 最小验收用例

---

## 禁止事项

- 因“GitHub 热门”直接并入主分支
- 无回退方案即进入核心链路
- 未经过验证即批量接入多个外部项
