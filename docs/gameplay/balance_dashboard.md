# 平衡看板使用指南

本指南用于说明如何使用 HUD 内置的平衡看板，帮助单人开发者快速做技能平衡迭代。

## 1. 看哪里

- 打开 HUD 的 `伤害统计` 面板。
- 关注 `施法效率榜` 区块。
- 该区块包含：
  - 榜单范围（`总榜` / `近60秒`）
  - 排序维度（`施放` / `效率` / `均命中`）
  - 样本过滤（`样本>=N`）
  - Top3 技能条目
  - 一行自动诊断结论

## 2. 指标含义

- `施放`：在当前统计范围内的施法次数。
- `均命中`：每次施法的平均命中数。
- `均时长`：每次施法的平均持续时长（毫秒）。
- `效率`：每秒平均命中数，计算公式：
  - `efficiency = avg_hits / max(avg_duration_ms / 1000, 0.001)`

## 3. 面板按钮

- `效率榜: 总榜 / 近60秒`
  - `总榜`：看整局构筑的长期表现。
  - `近60秒`：看最近改动后的即时反馈。
- `排序: 施放 / 效率 / 均命中`
  - `施放`：看使用频率。
  - `效率`：看输出吞吐能力。
  - `均命中`：看单次施法稳定性。
- `样本: >=1 / >=3 / >=5`
  - 过滤低样本噪声，防止偶然值误导。
- `实验模式一键设置`
  - 快速切到推荐口径：`近60秒 + 排序效率 + 样本>=3`。

### 快捷键（HUD调参）

- `F6`：导出效率快照
- `F7`：实验模式一键设置
- `F8`：战斗事件过滤（当前序列/全部序列）
- `F9`：对比最近两次
- `F10`：清空快照日志
- `F11`：开始新实验批次（清日志 + 一键实验模式）

面板底部会显示快捷键提示行，方便战斗中快速确认按键。

面板还会显示 `快照状态` 行，用于提示最近导出时间或是否已清空日志。

## 4. 自动诊断规则

诊断行基于榜单头部数据自动生成：

- 无有效条目：`暂无有效样本`。
- 若 Top1 相比 Top2 的效率显著领先：
  - `top1_eff / top2_eff >= overpower_efficiency_ratio`
  - 且 Top1 样本达到当前样本阈值
  - 判定为 `潜在过强`。
- 若样本不足：`样本偏少`。
- 否则：`当前分布健康`。

## 5. 配置文件

路径：`assets/config/hud_skill_efficiency.json`

```json
{
  "overpower_efficiency_ratio": 1.8,
  "default_min_casts": 3
}
```

- `overpower_efficiency_ratio`
  - 值越高：越严格，只有明显超模才提示“潜在过强”。
  - 值越低：越敏感，更容易触发预警。
- `default_min_casts`
  - 面板初始样本阈值。

## 6. 推荐调参流程（每轮）

1. 选定一个目标技能和一个对照技能。
2. 切到 `近60秒`，排序选 `效率`，样本设为 `>=3`。
3. 以稳定操作打 1-2 分钟。
4. 查看 Top3 与诊断结论。
5. 若目标技能过强：
   - 先降命中可靠性/覆盖，再降伤害或加 CD。
6. 若目标技能偏弱：
   - 先补命中可靠性，再补伤害。
7. 切回 `总榜`，确认长期影响是否合理。
8. 重复直到“数值表现”和“手感体感”都稳定。

## 7. 阈值建议

- 原型期（快速试错）：
  - `overpower_efficiency_ratio = 1.6`
  - `default_min_casts = 1 或 3`
- 中期制作（常规迭代）：
  - `overpower_efficiency_ratio = 1.8`
  - `default_min_casts = 3`
- 发版前收口（更稳健）：
  - `overpower_efficiency_ratio = 2.0`
  - `default_min_casts = 5`

## 8. 注意事项

- 看板用于辅助决策，不是唯一真值。
- 最终判断请结合三件事：
  - 数据（效率榜）
  - 可读性（警示与打击反馈是否清楚）
  - 手感（风险收益与操作负担）

## 8.5 一页式日常流程（5分钟版）

以下流程可作为每次调参后的固定动作：

1. `清空快照日志`（开启新实验批次）。
2. 点击 `实验模式一键设置`（自动固定为推荐口径）。
3. 导出一次“改动前”快照。
4. 改 1-2 个参数（只改一组，避免变量太多）。
5. 用相同操作强度打 60-120 秒。
6. 导出一次“改动后”快照。
7. 点击 `对比最近两次`：
   - 若出现 `口径不一致`，先按建议对齐后重做步骤 3-7。
8. 看结论并决策：
   - `潜在过强`：先减命中可靠性/覆盖，再降伤害或加 CD。
   - `样本偏少`：提高样本或延长测试时长。
   - `分布健康`：转入可读性与手感微调。

### 白盒试验场（第3周验证）

为了更稳定地复现“技能在固定环境下的手感/可读性”，可直接运行 `res://scenes/modules/skills/SkillsWhitebox.tscn`：

- 教学：单体（练起手/走位/收束）
- 组合：小群（练控场/切角/压制）
- 考核：30 秒刷怪（练持续输出与压迫下的可读性）
- 建议流程：先 `F11` 开新批次 → 模式结束后 `F6` 导出 → `F9` 对比

也可以用启动参数直接进入白盒场景：

- `--whitebox`：跳过菜单，直接进入技能白盒试验场
- `--whitebox-stage=tutorial|combo|exam`：进入后自动切到对应模式
- `--whitebox-exit-after=秒数`：白盒场景达到指定秒数后自动退出（可用于脚本化批量验证）
- `--whitebox-auto-batch`：进入白盒后自动执行“开始新实验批次”
- `--whitebox-auto-export-end`：考核模式结束时自动导出效率快照
- `--whitebox-auto-compare-end`：考核模式结束时自动对比最近两次（从第 2 轮开始触发，避免首轮快照不足）
- `--whitebox-repeat=N`：考核模式自动连跑 N 轮（每轮约 30 秒），用于批量采样
- `--whitebox-exam-seconds=秒数`：设置考核每轮时长（默认 30 秒，最小 5 秒）
- `--whitebox-seed=整数`：固定随机种子，提升重复测试的可比性
- `--whitebox-tag=标签`：为本次实验写入自定义标签（会记录到 round marker 与 summary）
- `--whitebox-pressure=low|mid|high`：设置考核压力档位（影响刷怪数量与刷怪间隔）
- `--whitebox-profile=quick|standard|stress`：白盒预设参数包（自动补齐 stage/repeat/时长/压力）

预设建议：

- `quick`：快速回归（默认约 20s * 2 轮，低压）
- `standard`：常规调参（默认约 30s * 3 轮，中压）
- `stress`：高压验证（默认约 45s * 5 轮，高压）

说明：`--whitebox-exit-after` 以白盒场景**总运行时长**计时（不会因轮次切换重置）。

白盒考核自动连跑时，会在 `balance_snapshots.log` 追加轮次标记：

- 会话开始：`tag=whitebox_session_start`（记录本次参数快照）
- `tag=whitebox_round_start`
- `tag=whitebox_round_end`
- 连跑结束时追加 `tag=whitebox_summary`
- 会话结束：`tag=whitebox_session_end`（记录结束原因、总耗时、完成轮次；中途退出会写 `reason=interrupted`）

并包含 `round`、`round_total`、`exam_seconds`、`seed` 等字段；`whitebox_summary` 还会包含 `enemy_avg`、`enemy_peak`，以及效率榜 Top1（`top1_skill`、`top1_eff`、`top1_casts` 等），便于快速判断“压力水平 + 输出表现”。

## 9. 快照导出与复盘

效率榜支持按钮 `导出效率快照`，会把当前面板状态写入：

- `user://balance_snapshots.log`

每条快照包含：

- 统计范围（总榜/近60秒）
- 排序维度（施放/效率/均命中）
- 样本阈值
- Top3 技能条目（施放、均命中、均时长、效率）
- 自动诊断结论
- 对比按钮触发后的结论摘要（会追加到同一日志）

### 推荐复盘模板（每轮调参后记录）

1. 本轮改动内容（改了什么参数）
2. 快照模式（范围/排序/样本）
3. Top1 与 Top2 的效率差
4. 诊断结论是否变化
5. 下一轮动作（削弱/补强/不动）

### 对比建议

- 同一技能只在同一模式下比较（例如都用 `近60秒 + 排序效率 + 样本>=3`）。
- 若结论抖动大，先提高样本阈值再判断。

### 最近两次对比（跨重启）

面板按钮 `对比最近两次` 具备两级数据来源：

1. 优先使用当前会话内存中的最近两次快照（实时、最快）。
2. 若当前会话快照不足，会自动回退读取 `user://balance_snapshots.log` 的最后两条快照再对比（跨重启可用）。

对比输出包含：

- 前后 Top1 技能名
- 前后 Top1 效率值
- 效率差值（Δ）
- 诊断是否变化

若提示“请先至少导出两次效率快照”，表示日志中可解析的有效快照数量不足，需要先导出两次。

### 日志维护

面板提供 `清空快照日志` 按钮：

- 会清空 `user://balance_snapshots.log`
- 同时清空当前会话的最近两次内存快照

适合在开启新一轮平衡实验前执行，避免不同实验批次互相干扰。

使用 `开始新实验批次`（或 `F11`）时会生成批次ID（例如 `B20260429-053700-123`）：

- 快照日志会记录批次ID
- 对比结论也会记录批次ID
- 对比日志会同时记录 `batch_prev` / `batch_last`（对比的两条快照各自的批次号），便于复盘时还原来源
- 快照/对比日志会附带 `batch_start_unix` 与 `batch_start_ms`（批次起点时间），用于后续按批次切片、重放或回溯时定位范围
- 面板状态行会显示当前批次ID
 - 可用 `批次: 全部 / 当前` 切换效率榜与快照统计范围；选择 `当前` 时仅统计本批次开始后的数据（若使用 `对比最近两次`，也会自动优先选择同批次快照）

若 `对比最近两次` 发现两条快照来自不同批次，会提示 `跨批次对比`，建议仅在同一批次内做参数效果判断。

若提示“口径不一致”，说明两次快照的以下任一项不同：

- 统计范围（总榜/近60秒）
- 排序维度（施放/效率/均命中）
- 样本阈值（>=1/3/5）

系统会在提示里附带建议动作（例如“范围统一为近60秒”），按建议统一后再导出新快照比较。

### 标准对比清单（避免误判）

1. 先固定口径（范围/排序/样本阈值）。
2. 导出“改动前”快照。
3. 调整参数并进行同等时长测试。
4. 导出“改动后”快照。
5. 点击 `对比最近两次`。
6. 若出现“口径不一致”，先按建议对齐，再重复步骤 2-5。
